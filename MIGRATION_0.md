# Migrating from Pre-Phase-0 Hula

Phase 0 is a substantial architectural rewrite. This document captures every
user-visible break and how to adapt.

## TL;DR

- **All admin APIs moved from Fiber HTTP to gRPC + REST gateway on `/api/v1/*`.** The legacy `/api/*` paths still work (transitional bridge); they'll be removed after Stage 0.8 closes.
- **Fiber is gone.** `handler/fiber_adapter.go`, `router/router.go`, `middleware/*`, `server/run.go` all deleted. `backend/proxy.go` rewritten on `httputil.ReverseProxy`. `main.go` calls `server.RunUnified(ctx, cfg)` now.
- **One unified HTTPS listener.** gRPC, REST gateway, WebDAV, visitor tracking, scripts, `/hulastatus`, and per-host backend proxies all share a single listener.
- **ClickHouse `events` schema expanded** with 18 enrichment columns (session_id, channel, UTM fields, browser/OS/device, country/region/city, etc.). GORM AutoMigrate handles existing deployments; a versioned migration runner exists for future explicit DDL.
- **BoltDB (`pkg/store/raft`)** is a new persistent store for user / RBAC / sessions / tokens. Always compiled in; not yet wired into startup in Phase 0 — the legacy ClickHouse user table remains in use for Phase 0 auth.
- **New auth provider framework** — Google / GitHub / Microsoft OIDC plus internal password + TOTP. Configured under `auth.providers:` in `config.yaml`.

## Breaking changes

### 1. `hula_ssl` is now **required**

The unified HTTPS listener needs a cert + key on disk. There's no in-process self-signed fallback in this commit.

```yaml
hula_ssl:
  cert: /etc/hula/certs/server.crt
  key:  /etc/hula/certs/server.key
```

ACME and per-host SNI cert selection are planned for a targeted follow-up on the unified server; until then, use static certs (including the ones generated by `mkcert` or `hulactl selfcert`).

### 2. API paths moving to `/api/v1/*`

Every admin API is available in three forms:

| Surface | Base path | Notes |
|---------|-----------|-------|
| Native gRPC | — | Dial on the same HTTPS port; use generated stubs from `pkg/apispec/v1/`. |
| REST gateway | `/api/v1/<service>/...` | JSON-over-HTTPS, auto-generated from the proto. |
| Legacy bridge | `/api/<service>/...` | Same handlers as pre-Phase-0. Deleted once hulactl and the e2e harness migrate. |

Hulactl still points at the legacy paths. No migration required for existing hulactl installations; they keep working until Stage 0.8 cuts over.

### 3. `config.yaml` additions

New top-level blocks (both optional):

```yaml
auth:
  providers:
    - name: google
      provider: oidc
      config:
        display_name: Google
        discovery_url: https://accounts.google.com/.well-known/openid-configuration
        client_id: ${HULA_GOOGLE_CLIENT_ID}
        client_secret: ${HULA_GOOGLE_CLIENT_SECRET}
        redirect_url: https://${HULA_HOST}/api/v1/auth/callback/google
        scopes: [openid, email, profile]
    # repeat for github, microsoft …

analytics:
  events_ttl_days: 395   # default; ~13 months of raw events
```

### 4. `config.yaml` fields added

On the top-level `Config`:

- `hostname` — alias for `hula_host`, used by adopted izcr code.
- `registry_hostnames` — comma-separated; unused in hula v1, reserved.

### 5. Imports and package paths

The following packages are gone:

- `github.com/gofiber/fiber/v2`
- `github.com/valyala/fasthttp`
- `github.com/tlalocweb/hulation/middleware` (all three files deleted)
- `github.com/tlalocweb/hulation/router` (deleted)
- `github.com/tlalocweb/hulation/handler` — `fiber_adapter.go` and several Fiber-specific helpers removed; net/http adapter is now the sole request-context abstraction.

New packages under `pkg/`:

- `pkg/apispec/v1/{auth,forms,landers,site,staging,badactor,status,analytics}` — gRPC service definitions.
- `pkg/apiobjects/v1` — shared message types (User, RBAC primitives, Session, Token).
- `pkg/server/{unified,grpc,http,static}` — the unified server.
- `pkg/server/authware` — OPA-policy-driven auth middleware.
- `pkg/server/authware/provider/{base,internal,oidc}` — auth provider framework.
- `pkg/server/authware/access` — server-access ACL helpers (scope="server" RoleAssignments).
- `pkg/server/authware/tokens` — JWT factory.
- `pkg/store/{common,raft}` — pluggable Storage interface + RaftStorage (BoltDB).
- `pkg/store/clickhouse` — ClickHouse schema + migration runner (pre-packaged `events_v1.sql`, `mv_events_hourly/daily/sessions.sql`).
- `pkg/analytics/enrich` — referrer classification, UTM parsing, UA decoding, session-id derivation.
- `pkg/tune` — runtime tunables (raft timeouts, etc.).

## No-op migrations (still works)

- **WebDAV PATCH** (X-Update-Range / X-Patch-Format: diff) behaves identically; handler just moved from Fiber to the unified ServeMux.
- **Visitor tracking** (`/v/*`) — unchanged.
- **Hugo / Docker / git-autodeploy** — unchanged.
- **Badactor signatures** — unchanged.
- **ClickHouse** — continues to use GORM AutoMigrate for backward compat; the new events_v1 explicit DDL exists in `pkg/store/clickhouse/schema/` but isn't applied yet (flip the switch in a focused follow-up).

## Deferred to Phase 1

- Bolt-user-store migration (replaces the ClickHouse `users` table for auth/RBAC).
- Email dispatch integration (invite flow, password-reset flow land after this).
- `GrantServerAccess` / `RevokeServerAccess` / `ListServerAccess` RPC implementations.
- ACME + per-host SNI cert selection on the unified listener.
- Analytics query APIs (Summary, Timeseries, Pages, Sources, Geography, Devices, Events, Forms, Visitors, Realtime). The proto skeleton is already in place; implementations are Phase 1.
- Svelte + shadcn UI that consumes the analytics APIs.

## Rolling back

Phase 0 is a forward-only migration. The Fiber-based listener is deleted; there's no config flag to re-enable it. Rollback means checking out the pre-Phase-0 branch (everything before `1070870`).

Data in ClickHouse is backward-compatible — legacy rows in the `events` table have empty values for the new enrichment columns and query correctly against the legacy schema. A future `events_v1` rename-and-migrate is non-destructive (old table kept as `events_legacy_v0` for 30 days).

## Quick-start for an existing deployment

1. Update `config.yaml`:
   - Add `hula_ssl: {cert, key}` if not already set.
   - Optional: add `auth.providers` for SSO.
2. Pull the new hula binary and restart.
3. Verify: `curl --http2 -v https://$HULA_HOST/hulastatus` should return 200 over HTTP/2; `grpcurl -H 'authorization: Bearer $TOKEN' $HULA_HOST:443 hulation.v1.status.StatusService/Status` should return `ok: true`.
4. Optional: list services via reflection — `grpcurl $HULA_HOST:443 list`.

## Known issues

- Pre-existing: `store/bolt.go` (the old hula store package, not the new `pkg/store/raft`) has undefined-type errors at HEAD. Unrelated to Phase 0. `go build .` works; `go build ./...` fails on the `store/` and `model/` (config-test) packages.
